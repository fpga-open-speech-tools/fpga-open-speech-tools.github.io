Audio Mini

Overview

The Audio Mini is an audio processing cape designed for the Altera DE-10 Nano. This cape contains an AD1939 audio codec that is used to sample audio from the line input and play back audio to the headphone output. The sampled audio is passed to a Cyclone V FPGA on the DE-10 Nano where the data can be processed. This platform is ideally suited for educational use as it is relatively low-cost and can be used in conjunction with Mathworks Simulink and FPGA Open Speech Toolsâ€™ code generation tool to rapidly create effects. This workflow also makes the Audio Mini an excellent tool for developers who wish to create prototype algorithms for devices such as hearing aids or sound effects processors.

Demos

Passthrough

The pass through project is a simple design where audio is passed directly from line in to headphone out; no processing is performed inside the FPGA. This makes the pass through project an ideal reference design for development because all other systems of the Audio Mini are functional and custom processing components can be easily connected.

Simple Filter Bank

The simple filter bank is an example design that uses a graphical user interface to modify the amplitudes of certain frequency ranges of an audio signal. The filter bank is made up of two channels, each containing a master volume and four bandpass filters implemented in the FPGA fabric. These elements are controlled through loadable kernel modules from Linux running on the hard processor system of the FPGA.

Multi-Effects Processor

The multi-effects processor is an example design that contains three distinct sound effects; a bitcrusher, an echo, and a flanger. The bitcrusher truncates a 32-bit audio word, creating noticable quantization noise when a sufficient number of bits have been eliminated. The echo effect combines a time shifted copy of an audio signal with itself. The flanger, similar to the echo, also combines a time shifted copy of an audio signal with itself, however this copy does not have a constant delay. Instead, the flanger uses a low frequency oscillator to modulate the delay of the signal copy. Each effect can be controlled through a web-based application, via sliders and switches, that accesses various sound effect registers in the FPGA fabric through loadable kernel modules.
